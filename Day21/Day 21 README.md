#  Day 21 — Object-Oriented Programming (Classes & Objects)

Today I learned the basics of **Object-Oriented Programming (OOP)** in Python.  
Classes act as **blueprints**, and objects are **real-world instances** created from those blueprints.  
This makes my systems more **organized, reusable, and scalable**.



##  Projects I Built

###  Iqama Records (Class & Objects)
Created an `Iqama` class with attributes (name, year, fee) and a method to display records.

###  VAT Invoices (Class & Methods)
Built an `Invoice` class with attributes and a method to calculate totals with VAT.

###  Housing Records (Annual Rent Calculation)
Designed a `Housing` class with a method to calculate annual rent.

###  Student Grades (Status Method)
Implemented a `Student` class with a method to determine grade status.

###  Shopping Cart (Subtotal Calculation)
Created a `CartItem` class with attributes and a method to calculate subtotal.

##  Reflections
- Classes are **blueprints**; objects are **instances**.  
- Methods make objects **active** (they can calculate, display, etc.).  
- OOP makes my systems **professional and scalable**.  
- My repo now shows **real applications with OOP design**.
