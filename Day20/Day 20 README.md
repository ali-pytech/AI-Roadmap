# Day 20 — File Handling in Python

Today I learned how to **read and write files** in Python. This allows my systems to save data permanently in text, CSV, and JSON formats. Now my applications can store records and retrieve them later, just like real-world software.



## Projects I Built

### Iqama Records (Text File)
I wrote iqama records into a text file and read them back. This shows how simple records can be stored and retrieved.

### VAT Invoices (CSV File)
I saved invoices into a CSV file with headers. This is perfect for tabular financial data.

### Housing Records (JSON File)
I stored housing data in JSON format. JSON is great for structured records with multiple fields.

### Student Grades (Text File with Error Handling)
I wrote student grades into a text file and added error handling for missing files.

### Shopping Cart (JSON File)
I saved shopping cart items into a JSON file and read them back. This looks like a real e-commerce system.



##  Reflections
- Text files are simple for plain records.  
- CSV is ideal for tabular data like invoices.  
- JSON is best for structured records like housing and cart systems.  
- Error handling ensures files are managed safely.  
- My repo now looks like **real applications with persistent storage**.
